QuantX — System Overview

This page exists to preserve intent. It is a reference document for future maintainers, contributors, and automated assistants interacting with the QuantX project.

1. What QuantX Is

QuantX is a private, discipline-first sports betting decision framework. Its purpose is to enforce process integrity around wagering decisions.

QuantX does not attempt to predict outcomes, select winners, or optimize betting performance. All wagers are executed manually by the user at sportsbooks.

The system is designed to constrain behavior, not to encourage activity. Silence and inaction are valid and intentional outcomes.

2. What QuantX Is Not

QuantX is explicitly not:

  • A prediction engine
  • A betting bot or automated wagering system
  • A model-driven or statistical forecasting platform
  • An ROI or performance-tracking dashboard
  • A bankroll optimization tool (e.g., Kelly betting)
  • A gamified or competitive product

These exclusions are intentional design decisions, not missing features.

3. Core Principles (Frozen)

QuantX enforces the following principles mechanically:

  • No action without a signal
  • No signal without evaluation
  • No evaluation without explicit resolution
  • No bet sizing before PLAY
  • No loss chasing
  • No martingale logic
  • No confidence-based stake inflation
  • No automation creep

Silence is treated as a feature, not a failure mode.

4. High-Level System Architecture

QuantX operates as a closed, deterministic loop:

Signal → Watchlist → Evaluate → Decide → Stake → Cap → Exit → Silence

Each step is hard-gated. Skipping steps is intentionally impossible. The system prioritizes correctness of flow over speed or convenience.

5. Enforcement Philosophy

QuantX assumes that users (and browsers) will eventually test boundaries. As a result, enforcement is implemented at the system level, not solely in the UI.

Key enforcement decisions include:

  • Single-use signal tokens
  • Immediate token burn on evaluation
  • One-way decision paths
  • Explicit daily exposure caps
  • Mandatory day closeout

User interface cues (e.g., disabled buttons) are treated as non-authoritative. System state is the final authority.

6. Explicitly Out of Scope

The following areas are intentionally excluded from QuantX:

  • Automated bet placement
  • Predictive modeling
  • Outcome tracking or ROI reporting
  • Parlays or exotic wagers
  • Performance gamification

Proposals to add these features should be considered violations of the system’s purpose.

7. Version Status

QuantX v1 is complete.

Core logic and enforcement mechanisms are considered frozen. Changes are justified only to correct genuine workflow flaws or enforcement gaps.

Improvements to presentation, clarity, or contextual awareness belong to subsequent phases.

8. Intended Future Work (Non-Binding)

Future work may include:

  • User interface calm and visual unification
  • Platform hygiene and deployment refinement
  • A contextual, non-predictive information layer (e.g., league state awareness)

Any future additions must adhere to the guiding principle:

Add clarity without adding pressure. Add awareness without adding edges.

QuantX exists to prevent misuse, not to maximize engagement. If no action is taken, the system is working as intended.
